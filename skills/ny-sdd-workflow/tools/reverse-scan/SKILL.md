# Reverse Scan — 深度业务代码扫描 Skill

## 定位

为没有 Spec 体系的遗留项目建立完整的**代码认知层**，使后续编码能精准复用已有的业务逻辑、函数方法、中间件封装和架构模式。

产出物与正向 Spec 体系（`.project/specs/master/`）路径隔离，不污染 index.md。

## 输入前提

本 Skill 被调用时，以下信息应已由调用方准备就绪：

| 前提 | 说明 |
|------|------|
| project-profile.md 已有技术栈、外部依赖 | 基础扫描已完成 |
| frontend-context.md / backend-context.md 已生成 | 前后端规范已提取 |
| .docs/ 已扫描（如有文档） | 文档已处理 |

如以上前提未就绪，Skill 在对应步骤中自行补全。

## 执行指令

读取本 Skill 目录下的 `rules/scan-workflow.md`，按其中的流程逐步执行。

模板文件位于 `rules/card-template.md`，在生成知识卡片、模块地图、DES、REQ 时读取对应模板。

## 产出物清单

执行完毕后，`.project/reverse-scan/` 目录下必须存在以下文件：

### 认知层产出

| 文件路径 | 说明 |
|---------|------|
| `knowledge-cards/*.md` | 逐文件知识卡片（函数级业务语义、调用关系、数据操作） |
| `call-graph.md` | 全局调用关系图 + 被多方调用的关键节点 |
| `module-map.md` | 模块地图（已有模块划分 + 职责 + 依赖关系） |
| `db-schema.md` | 完整数据库结构（表/字段/类型/约束/索引/外键） |

### Spec 快照（已有模块的逆向 Spec，仅供参考）

| 文件路径 | 说明 |
|---------|------|
| `specs/requirements/REQ-{xx}-{name}.md` | 已有模块的需求规格（从代码反推） |
| `specs/design/DES-{xx}-{name}.md` | 已有模块的设计规格（从代码反推） |

### 过程与验证

| 文件路径 | 说明 |
|---------|------|
| `verification-report.md` | 全局交叉验证报告 |
| `grey-decisions.md` | 灰色决策日志（所有暂停点的决策记录） |
| `scan-summary.md` | 扫描完成报告（统计信息 + 审计结果） |

### 合并就绪文件（供主工作流读取合并，本 Skill 不直接写入主工作流文件）

| 文件路径 | 内容 | 主工作流合并目标 |
|---------|------|----------------|
| `profile-patch.md` | 「业务架构」「业务流程」区块（自底向上涌现） | 由主工作流合并到 project-profile.md |
| `overview.md` | 项目全景文档一～七章（基于全量认知生成） | 由主工作流合并到 .outdocs/project-overview.md |
| `api-doc.md` | 从 DES 提取的全量接口文档 | 由主工作流合并到 .outdocs/api-doc.md |
| `scan-summary.md` | 扫描完成报告（含统计信息 + 审计结果） | 由主工作流追加到 context.md |

## 产出物更新规则

- **首次调用**：旧项目初始化时执行一次，产出完整认知层
- **快照不可变**：本 Skill 产出为初始化时的快照，编码过程中不回写更新
- **全量重扫**：用户说"重新扫描" → AI 备份 `.project/reverse-scan/` 为 `.project/reverse-scan.bak/`，重新执行本 Skill

## 下游消费方式

本 Skill 的产出物供主工作流按需加载。主工作流自行决定何时加载哪些文件，本 Skill 不感知主工作流的阶段编号。

**产出物 → 消费场景映射**：

| 产出物 | 典型消费场景 | 用途 |
|--------|-------------|------|
| module-map.md | 任务拆分 | 判断新需求归入已有模块还是新建模块 |
| 知识卡片 | 需求分析 / 方案设计 / 编码 / 审计 | 精确界定新旧边界、精准复用已有方法 |
| call-graph.md | 方案设计 / 影响面评估 | 确保新 API 不重复、评估变更波及范围 |
| db-schema.md | 方案设计 / 文档学习 / 影响面评估 | 确认表结构和字段约束、评估数据关联 |
| reverse-scan REQ | 需求分析 | 了解已有功能和验收标准 |
| reverse-scan DES | 方案设计 / 编码 | 了解已有技术方案和核心流程 |
| profile-patch.md | 初始化合并 | 业务架构/流程写入 project-profile.md |
| overview.md | 初始化合并 | 项目全景文档 |
| api-doc.md | 初始化合并 | 全量接口文档 |

**条件加载规则**：`.project/reverse-scan/` 存在时加载，不存在时跳过（主工作流走兜底）。
