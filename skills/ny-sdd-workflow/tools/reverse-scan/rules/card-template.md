# 产出物模板

> 本文件定义 reverse-scan Skill 各产出物的格式标准。生成对应产出物时读取此文件。

---

## 1. 知识卡片模板

文件路径：`.project/reverse-scan/knowledge-cards/{文件路径的扁平化命名}.md`

扁平化命名规则：将文件路径中的 `/` 替换为 `--`，去掉文件扩展名。
示例：`src/service/ReportService.java` → `src--service--ReportService.md`

```markdown
# 知识卡片: {文件路径}

## 文件信息
- **路径**: {完整相对路径}
- **角色**: {Controller / Service / DAO-Mapper / Entity / 前端页面 / 前端组件 / Store / API调用层 / 其他}
- **关联表**: {该文件操作的数据库表列表，无则填"无"}

## 函数清单

### {函数签名}
- **业务语义**: {用一句业务语言描述，如"根据报告ID查询解析结果并返回"，不是"调用mapper查数据库"}
- **输入**: {参数列表：名称 / 类型 / 约束 / 可选性}
- **输出**: {返回值：类型 / 结构 / 可能的值域}
- **核心逻辑**:
  1. {步骤1，业务语言描述}
  2. {步骤2}
  3. ...
- **调用了**: {类名.方法名 列表}
- **被谁调用**: {类名.方法名 列表，跨文件调用在 Step R1.3 补充}
- **操作的数据**: 读 {表.字段}; 写 {表.字段}
- **异常处理**: {异常场景 → 处理方式}
- **状态变更**: {实体.状态字段: 旧值 → 新值, 条件: ...}（无则填"无"）
- **权限要求**: {角色/注解}（无则填"无"）
- **可疑标记**: {无 / dead code / magic number=XX / workaround / TODO / FIXME / ...}

### {下一个函数...}
```

**前端页面额外维度**（追加到对应文件的知识卡片末尾）：

```markdown
## 页面信息
- **路由**: {路由路径，含参数}
- **用途**: {用业务语言描述这个页面让用户做什么}
- **组件结构**: {包含的子组件列表 + 布局方式}
- **用户交互**:
  | 交互元素 | 类型 | 触发操作 |
  |---------|------|---------|
  | {元素名} | {按钮/表单/列表/弹窗/...} | {调用什么函数/API/跳转到哪} |
- **API 调用**:
  - 页面加载: {API 列表}
  - 用户操作: {操作 → API 映射}
- **页面跳转**:
  | 目标页面 | 触发条件 |
  |---------|---------|
  | {路由} | {什么情况下跳转} |
```
